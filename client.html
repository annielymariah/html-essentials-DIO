O que é arquitetura cliente-servidor?

(Voltar)

A arquitetura cliente-servidor é um modelo de comunicação em rede onde um dispositivo (o cliente) solicita serviços ou recursos a outro dispositivo (o servidor), que os processa e responde ao cliente. Nesse modelo, o cliente é responsável por enviar requisições e o servidor é responsável por processá-las e fornecer a resposta. O servidor geralmente armazena dados ou executa operações que o cliente não pode realizar localmente.

O que é um Cliente

No contexto da arquitetura Cliente-Servidor, o cliente é um dispositivo, software ou aplicação que solicita serviços ou recursos de um servidor. Ele atua como a interface do usuário ou ponto de acesso que permite que os usuários finais interajam com o sistema. O cliente pode ser um computador, smartphone, tablet ou qualquer outro dispositivo capaz de se comunicar com um servidor.

Funções e Características do Cliente

(Voltar)
  • Solicitação de Serviços: O cliente é responsável por fazer as requisições que o servidor deve processar, como acessar uma página web ou realizar um login.
  • Interface com o Usuário: O cliente oferece a interface gráfica ou de interação com o usuário, como navegadores ou aplicativos móveis.
  • Processamento Local: O cliente pode realizar alguns processamentos locais, como validação de entrada de dados ou renderização de interfaces gráficas.
  • Armazenamento Local: O cliente pode armazenar dados temporários como cookies ou cache.
  • Comunicação com o Servidor: O cliente estabelece a comunicação com o servidor através de protocolos como HTTP ou FTP.
  • Consumidor de Recursos: O cliente consome os serviços ou dados fornecidos pelo servidor, como streaming de vídeo ou áudio.
  • Exemplo de Clientes: Navegadores Web (Chrome, Firefox), Aplicativos Móveis (Instagram, WhatsApp), Software de e-mail (Outlook, Thunderbird), Softwares de FTP (FileZilla, WinSCP).

Tipos de Clientes

(Voltar)
  • Cliente Pesado (Fat Client): Possui grande parte do processamento e armazenamento local. Exemplos: Photoshop, Microsoft Word.
  • Cliente Leve (Thin Client): Depende quase completamente do servidor para processamento. Exemplos: Terminais de acesso, aplicativos web acessados via navegador.

Características de Comunicação Cliente-Servidor

(Voltar)
  • Requisição e Resposta: O cliente envia uma requisição ao servidor, que responde com os dados solicitados.
  • Interatividade: A comunicação é muitas vezes interativa, permitindo que o cliente envie dados para o servidor e receba feedback em tempo real.
  • Protocolos de Comunicação: O cliente e o servidor se comunicam através de protocolos como HTTP, HTTPS, FTP ou WebSocket.

Exemplos de Como um Cliente Funciona

(Voltar)
  • Navegador Web: Quando você acessa um site, o navegador envia uma requisição HTTP para o servidor, que responde com os dados da página.
  • Aplicativos de e-mail: O cliente de e-mail envia uma solicitação ao servidor de e-mail para buscar ou enviar mensagens.

Preocupações do Lado do Cliente

(Voltar)
  • Tipos de resoluções de tela dos dispositivos (Responsividade).
  • Navegadores utilizados (Compatibilidade, boas práticas e testes).

O que é o Server

(Voltar)

Um server (servidor) é um computador, sistema ou programa que fornece serviços, dados ou recursos a outros dispositivos, chamados clientes, por meio de uma rede, como a internet ou uma rede local.

Tipos de Servidores

(Voltar)
  • Data base Server (Arquivos): Armazena e gerencia arquivos compartilhados na rede.
  • Segurança (Firewall).
  • Streaming.
  • Servidor de Correio (E-mail): Gerencia o envio e recebimento de mensagens de e-mail.
  • Web Server: Responde a solicitações de navegadores e entrega páginas HTML, imagens e outros recursos.
  • Servidor Proxy: Intermediário entre o cliente e o servidor de destino.
  • Servidor DNS: Traduz nomes de domínio em endereços IP.

Data Centers

(Voltar)

Data centers são instalações físicas projetadas para abrigar servidores, sistemas de armazenamento, equipamentos de rede e outros componentes necessários para processar, armazenar e distribuir dados em larga escala.

Construção do Server (Hardware e Software)

(Voltar)

Hardware é a parte física de um sistema de computação, enquanto Software é a parte lógica que instrui o hardware. Ambos trabalham juntos para o funcionamento de dispositivos computacionais.

Exemplos de Servidores Web

(Voltar)
  • Apache HTTP Server: Servidor web de código aberto, amplamente utilizado.
  • Nginx: Servidor web de alto desempenho, conhecido pela baixa utilização de recursos.
  • Microsoft IIS: Servidor web da Microsoft, usado principalmente em sistemas Windows Server.
  • LiteSpeed: Focado em velocidade e otimização de recursos.
  • Tomcat: Servidor web para aplicações Java.
  • Caddy: Servidor web moderno com suporte integrado para HTTPS.
